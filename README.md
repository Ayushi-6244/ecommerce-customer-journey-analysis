# E-Commerce Customer Journey & Conversion Analysis

## Project Overview
This project analyzes customer journey data from an e-commerce platform to understand user behavior, identify checkout drop-offs, and uncover opportunities to improve conversion rates.

Using Python and data analysis techniques, the project explores how device usage, referral sources, engagement levels, and cart behavior influence purchasing outcomes.

## Dataset
The dataset contains over 12,000 customer interactions including:

- session ID
- device type
- referral source
- country
- time spent on page
- items added to cart
- purchase outcome

## Key Analyses

### Checkout Funnel Analysis
Identified major drop-off points across the customer journey:
Home → Product Page → Cart → Checkout → Confirmation

### Device Performance
Analyzed conversion rates across mobile, desktop, and tablet devices.

### Referral Source Performance
Compared marketing channels such as Google, Email, Social Media, and Direct traffic.

### Engagement Analysis
Examined the relationship between time spent on pages and purchase likelihood.

### Cart Behavior Analysis
Analyzed how the number of items added to the cart affects conversion probability.

### Time-of-Day Analysis
Explored user activity patterns and peak purchasing hours.

## Tools Used
- Python
- Pandas
- Matplotlib
- Seaborn
- Jupyter Notebook

## Business Insights
The analysis revealed several opportunities to improve conversion rates:

- Improve product page experience to reduce drop-offs before cart stage
- Optimize mobile checkout experience
- Focus marketing investment on high-performing referral channels
- Encourage cart engagement through product recommendations
- Schedule promotions during peak activity hours

## Conclusion
This project demonstrates how behavioral data analysis can help e-commerce companies optimize the customer journey and improve conversion performance.
